Types of Animations

Animation

An animation lets an element gradually change from one style to another. You can change as many CSS properties you want, as many times you want. To use CSS animation, you must first specify some keyframes for the animation.

Animation Delay

Animation Delay

The animation-delay CSS property specifies the amount of time to wait from applying the animation to an element before beginning to perform the animation. The animation can start later, immediately from its beginning, or immediately and partway through the animation.

Code

div {
width: 100px;
height: 100px;
background: yellow;
position: relative;
animation: animation1 5s infinite;
animation-delay: 5s;
}
@keyframes animation1{
from {left: 0px;}
to {left: 200px;}
}

Animation Duration

Animation Duration

The CSS animation-duration property allows you to specify how long your animation will take to complete one cycle. ... For example, a value of 3s will result in an animation running for 3 seconds. By changing the value of the animation-duration property, you effectively change the speed at which your animation runs.

Code

div {
width: 100px;
height: 100px;
background: yellow;
position: relative;
animation: animation3 5s infinite;
animation-duration: 5s;
}
@keyframes animation3{
from {left: 0px;}
to {left: 200px;}
}

Animation should complete one cycle in 5 seconds:

Animation Iteration Count

Animation Iteration Count

The animation-iteration-count CSS property sets the number of times an animation sequence should be played before stopping. If multiple values are specified, each time the animation is played the next value in the list is used, cycling back to the first value after the last one is used.

Code

div {
width: 100px;
height: 100px;
background: yellow;
position: relative;
animation: animation4 3s;
animation-iteration-count: 5;
}
@keyframes animation4{
from {left: 0px;}
to {left: 200px;}
}

Play the animation 5 times:

Animation Fill Mode

Animation Fill Mode

The animation-fill-mode property specifies a style for the element when the animation is not playing (before it starts, after it ends, or both). CSS animations do not affect the element before the first keyframe is played or after the last keyframe is played. The animation-fill-mode property can override this behavior.

Code

div {
width: 100px;
height: 100px;
background: yellow;
position: relative;
animation: animation5 3s;
animation-fill-mode: forwards;
}
@keyframes animation5{
from {left: 0px;}
to {left: 200px;background-color: blue;}
}

Let the div element retain the style values from the last keyframe when the animation ends:

Animation Fill Mode(Backwards)

Code

div {
width: 100px;
height: 100px;
background: yellow;
position: relative;
animation: animation51 3s;
animation-fill-mode: backwards;
}
@keyframes animation51{
from {left: 0px;}
to {left: 200px;background-color: blue;}
}

Let the div element get the style values set by the first keyframe before the animation starts (during the animation-delay period):

Animation Fill Mode(Both)

Code

div {
width: 100px;
height: 100px;
background: yellow;
position: relative;

animation-fill-mode: both;

}

Let the div element get the style values set by the first keyframe before the animation starts, and retain the style values from the last keyframe when the animation ends:

Animation Timing Function

Animation Timing Function

Code

height: 100px;
background: yellow;
position: relative;
animation: animation7 3s;
animation-timing-function: linear; }
@keyframes animation7{
from {left: 0px;}
to {left: 200px;}
}

Play an animation with the same speed from beginning to end:
